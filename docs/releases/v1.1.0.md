# v1.1.0 — Server Scheduled Reminders

## Summary

Bristol Care Dashboard v1.1 adds server-side scheduled push notifications via Vercel Cron, enabling automatic daily reminders for weather, deadlines, miss-you countdowns, and period predictions. A new admin monitoring dashboard provides visibility into cron execution, delivery statistics, and manual dry-run testing.

## Added

- **Server-scheduled reminders** (`/api/cron/reminders`): Vercel Cron triggers daily at UTC 09:00. Generates and sends Push notifications for weather, deadline, miss-you, and period reminders based on per-space preferences.
- **Reminder preferences** (`reminder_preferences` table): Stores reminder time, enabled types, and timezone in Supabase. Synced from the settings page.
- **Delivery deduplication** (`reminder_delivery_log` table): Ensures each reminder type is sent only once per space per day.
- **Run logs** (`reminder_run_logs` table): Records every cron execution with duration, skipped reasons, and error summaries.
- **Admin monitoring tab**: New "提醒监控" tab in the data maintenance center showing configuration status, 7-day statistics, recent run history, diagnostic hints, and a dry-run button.
- **Dry-run API** (`/api/admin/reminders/dry-run`): Simulates a cron run without sending real push notifications. Writes a run log entry with trigger_type `manual_dry_run`.
- **Status API** (`/api/admin/reminders/status`): Returns recent run logs, aggregated statistics, and configuration status.
- **lib/serverReminderScheduler.ts**: Pure-function scheduler with time window validation and delivery log deduplication.
- **Inactive subscription cleanup**: Push subscriptions returning 410/404 are automatically marked inactive during cron runs.

## Changed

- `vercel.json`: Added `crons` configuration for daily UTC 09:00 trigger
- `/api/health`: Added `CRON_SECRET` status check
- `.env.example`: Added `CRON_SECRET` variable with documentation
- `supabase/schema.sql`: Added three new tables (`reminder_preferences`, `reminder_delivery_log`, `reminder_run_logs`)

## Stability

- All cron endpoints are protected by `CRON_SECRET` Bearer authentication
- Unauthorized requests receive 401 and do not write any logs
- Supabase unavailability returns graceful 503 responses without crashes
- No secrets, push endpoints, or VAPID keys are exposed in APIs, logs, or UI
- Dry-run mode does not send real push notifications or write delivery logs

## Tests

- **656** Vitest tests passing (up from 615 in v1.0.0)
- New test files: `serverReminderScheduler.test.ts` (13), `cronAuth.test.ts` (13), `reminderMonitoring.test.ts` (12)

## Deployment Notes

- Requires `CRON_SECRET` environment variable in Vercel
- Requires Supabase schema migration (run `supabase/schema.sql`)
- Requires VAPID keys for push delivery (run `npm run generate:vapid`)
- Cron time is UTC — adjust `reminder_time` preferences accordingly
- See `docs/deployment.md` and `docs/maintenance.md` for full details
