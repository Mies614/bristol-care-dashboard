# S2.3 Private Media Read — Rollback

Run `20260626000007_rollback_private_media_buckets.sql`.

This restores `couple-albums` and `love-notes` to `public=true`.

## Effects

- Existing media becomes publicly readable again (same as pre-S2.3)
- Signed URLs continue to work (they're generated by service role regardless of bucket public state)
- Anonymous INSERT remains disabled (S2.2 hotfix is separate)

## Risk

Rollback re-exposes all media to public anonymous access. Anyone with the anon key and object path can read media.

Only use if signed-read code is being rolled back or if signed URL performance is unacceptable.
